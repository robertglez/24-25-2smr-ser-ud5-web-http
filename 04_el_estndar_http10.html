04| El estándar HTTP/1.0

El estándar HTTP/1.0 recoge únicamente tres órdenes que representan las operaciones de recepción, envío de información y revisión del estado.

  • Método de petición GET. Se utiliza para solicitar un recurso del servidor. Siempre que pulsamos sobre un enlace o escribimos una dirección de Internet en la barra de navegación de un navegador web, estamos utilizando este método de petición. Como resultado, el servidor HTTP envía el documento correspondiente a la dirección de Internet seleccionada.
  • Método de petición HEAD. Solicita información sobre un recurso, como su tamaño, tipo, fecha de modificación, etc. Suele ser utilizado por los gestores de caché de páginas o por los servidores intermedios (proxy servers) para determinar cuándo es necesario actualizar la copia almacenada de un archivo determinado.
  • Método de petición POST. Se utiliza para enviar información al servidor, como los datos contenidos en un formulario. El servidor pasará esta información a un proceso encargado de su tratamiento (generalmente, una aplicación del servidor). La operación que se llevará a cabo con la información proporcionada dependerá de la dirección de Internet (URL) utilizada, especialmente en los formularios.

Es importante señalar que un cliente web selecciona automáticamente las órdenes HTTP necesarias para recuperar la información para el usuario. Por lo tanto, al activar un enlace, siempre se ejecuta una operación GET para recuperar el documento correspondiente.

Tal como muestra la siguiente imagen, la comunicación entre el navegador y el servidor se lleva a cabo en dos etapas. En primer lugar, el navegador realiza una solicitud HTTP que, posteriormente, es procesada por el servidor, el cual responde con una respuesta HTTP.

Si analizamos con detenimiento la solicitud HTTP, encontramos un conjunto de líneas de texto que el navegador envía al servidor. En primer lugar, está la línea de solicitud, que especifica el tipo de documento solicitado, el método aplicado y la versión del protocolo utilizada. Esta línea está compuesta por tres elementos separados por un espacio: el método, la dirección web y el protocolo HTTP utilizado por el cliente.

A continuación, se encuentran los campos del encabezado de la solicitud, un conjunto de líneas opcionales que proporcionan información adicional sobre la solicitud y/o el cliente (navegador, sistema operativo, etc.). Cada una de estas líneas está formada por un nombre que describe el tipo de encabezado, seguido de dos puntos (:) y el valor del encabezado.

Finalmente, el cuerpo de la solicitud consiste en un conjunto de líneas opcionales separadas de las anteriores por una línea en blanco. Su propósito es permitir el envío de datos al servidor mediante un formulario durante la transmisión.

Un ejemplo de solicitud HTTP sería el siguiente:

GET http://www.pego.org HTTP/1.0
Accept : Text/html If-Modified-Since : Saturday, 19-May-2006 17:39:13 GMT
User-Agent : Mozilla/6.0 (compatible; MSIE 5.0; Windows XP)

En consecuencia, la solicitud HTTP recibiría una respuesta HTTP, es decir, un conjunto de líneas que el servidor enviará al navegador web. Al igual que la solicitud, la respuesta HTTP contiene ciertos elementos clave, como la línea de estado. Esta línea especifica la versión del protocolo utilizada y el estado de la solicitud en curso, proporcionando un código y un mensaje explicativo.

En cuanto a los campos del encabezado de respuesta, este es un conjunto de líneas opcionales que aportan información adicional sobre la respuesta y/o el servidor. Cada una de estas líneas consta de un nombre que define el tipo de encabezado, seguido de dos puntos (:) y el valor del encabezado.

El cuerpo de la respuesta contiene el documento solicitado. A modo de ejemplo, a continuación se muestra una respuesta HTTP:

HTTP/1.0 200 OK Date: 19 May 2006 17:40:12 GMT Server : Microsoft-IIS/2.0
Content-Type : text/HTML Content-Length : 1245 Last-Modified : 11 May 2006 08:29:13 GMT

En lo que respecta a los códigos de respuesta, estos son los códigos que se muestran cuando el navegador no puede acceder a la página web solicitada. La sintaxis de una respuesta HTTP incluye una línea de estado con una estructura fija que indica la versión, el código del error y un mensaje explicativo.

Los posibles códigos de estado se identifican con números de tres dígitos y se clasifican en cinco grupos: códigos informativos (1xx), éxito en la solicitud (2xx), redirección de la solicitud (3xx), errores generados en el cliente (4xx) y errores generados en el servidor (5xx).

Las respuestas más comunes son el código 200, que indica la confirmación de la petición, y el 404, que indica que el objeto solicitado no se encuentra disponible.

Algunos de los códigos de estado más habituales son los siguientes:

Successful 2xx

Esta clase de código de estatus indica que la petición del cliente fue recibida, comprendida y aceptada con éxito.

200 OK

La petición ha tenido éxito. La información devuelta con la respuesta depende del método usado en la petición, tal y como sigue:

GET: una entidad correspondiente al recurso solicitado se envía con la respuesta;

HEAD: la respuesta debe contener únicamente la información de cabecera y sin Entidad-Cuerpo;

POST: una entidad que describe o contiene el resultado de la acción.

Redirection 3xx

301 Moved Permanently

302 Moved Temporarily

304 Not Modified

Client Error 4xx

400 Bad Request

401 Unauthorized. La petición requiere autenticación de usuario.

403 Forbidden. El servidor entiende la petición, pero rechaza satisfacerla.
404 Not Found. El servidor no ha encontrado nada que concuerde con la URI de la petición.

Server Error 5xx

500 Internal Server Error. El servidor encontró una condición inesperada que impide satisfacer la petición.

502 Bad Gateway. El servidor, actuando como gateway o proxy, recibió una respuesta inválida del servidor superior al que accedió intentando satisfacer la petición.

503 Service Unavailable. Actualmente el servidor no es capaz de gestionar la petición debido a una sobrecarga o mantenimiento temporal del servidor.

Para más información, visita: https://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10

Línea de encabezado HTTP

Campo fundamental que define gran parte de la información que se intercambia entre los clientes y los servidores. Estas líneas permiten el envío de información descriptiva en la transacción y facilitan, por ejemplo, la autenticación o identificación de los usuarios.
