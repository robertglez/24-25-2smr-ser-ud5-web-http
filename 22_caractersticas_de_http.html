2.2. Características de HTTP

El protocolo de transferencia de hipertexto (HTTP) es un protocolo cliente-servidor bastante sencillo que articula los intercambios de información entre los clientes web y los servidores HTTP. HTTP fue desarrollado por el consorcio W3C y la IETF. Esta colaboración culminó en el año 1999 con la publicación de una serie de RFC, el más importante de los cuales fue el RFC 2616, que especificaba la versión 1.1.

Los Request for Comments, más conocidos por sus siglas RFC, son una serie de publicaciones del grupo de trabajo de ingeniería de Internet que describen diversos aspectos del funcionamiento de Internet y otras redes de computadoras, como protocolos, procedimientos, etc., así como comentarios e ideas sobre estos.

Desde el punto de vista de las comunicaciones, está soportado en los servicios de conexión TCP/IP y funciona de la misma manera que el resto de servicios propios de los entornos UNIX.

Técnicamente, un proceso servidor escucha en un puerto de comunicaciones TCP (por defecto, el 80) y espera las solicitudes de conexión de los clientes web. Una vez establecida la conexión, el protocolo TCP se encarga de mantener la comunicación y garantizar un intercambio de datos libre de errores.

El protocolo de transferencia de hipertexto se basa en operaciones simples de solicitud/respuesta. Cuando un cliente establece una conexión con un servidor y envía un mensaje con los datos de la solicitud, el servidor responde con un mensaje similar, que contiene el estado de la operación y su resultado posible. Todas las operaciones pueden adjuntar un objeto o recurso sobre el cual actúan; cada objeto web (documento HTML, archivo multimedia o aplicación CGI) es conocido por su localizador uniforme de recursos (URL, uniform resource locator). Los recursos pueden ser archivos, el resultado de la ejecución de un programa, una consulta a una base de datos, la traducción automática de un documento, etc.

HTTP es un protocolo sin estado, es decir, no guarda ninguna información sobre conexiones anteriores. El desarrollo de aplicaciones web con frecuencia necesita mantener estado. Para ello se utilizan las cookies, es decir, la información que un servidor puede almacenar en el sistema cliente. Esto permite que las aplicaciones web establezcan la noción de “sesión” y, a la vez, permite rastrear usuarios, ya que las cookies pueden almacenarse en el cliente durante un tiempo indeterminado.

Localizador uniforme de recursos (URL)

Generalmente, la vía de la que dispone un usuario para acceder a una página web o un objeto (archivo) es mediante su localizador uniforme de recursos (URL), que es una especie de dirección que indica la localización exacta de un documento en Internet.

La dirección que indica la localización exacta del documento se compone del nombre o la dirección de Internet (IP, Internet Protocol) del servidor y el camino relativo del documento dentro del servidor. También puede incluir otros aspectos como el puerto por el que se solicita el servicio (por defecto, se asume que el 80 es para el protocolo HTTP, el 21 para el protocolo FTP, etc.) y un nombre de usuario y una contraseña para aquellos documentos que requieran autenticación para acceder.
