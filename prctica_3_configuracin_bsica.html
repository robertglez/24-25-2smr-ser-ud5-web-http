Práctica 3: Configuración básica

Configuración básica de un nodo virtual

Objetivos

Los objetivos de la actividad son:

  • Conocer los parámetros básicos de configuración de Apache2.
  • Distinguir entre los diferentes niveles de configuración (Global, por defecto, nodo virtual) que tiene Apache2.
  • Conocer los diferentes tipos de registros del servidor web Apache2.
  • Entender el concepto de nodo virtual (virtual host).
  • Crear un nodo virtual

Supuesto

Para realizar la siguiente actividad lo haremos desde el siguiente supuesto: nuestro equipo servirá como servidor de alojamiento de varios sitios web. Por lo tanto, estructuraremos la carpeta que contiene los sitios web /var/www de forma que para cada sitio web dispondremos de una carpeta que contendrá tanto las páginas html como los registros del servidor (logs). Esta carpeta tendrá el nombre del dominio a publicar. La imagen siguiente representa la estructura de carpetas alojando dos sitios web.

/var/www
├── html
│   └── index.html
├── secure.xarxa00.lan
│   ├── log
│   └── public_html
└── www.xarxa00.lan
    ├── log
    └── public_html


Para cada sitio web se creará un usuario en el sistema para que pueda administrar el contenido del sitio web. El usuario tendrá el mismo nombre que el dominio (user_www) y solo tendrá acceso vía FTP a la carpeta del dominio (www.xarxa00.lan).

Actividad (parte 1): creación del sitio web virtual

  • El nuevo sitio web virtual responderá al nombre de dominio www.xarxa##.lan.
  • Siguiendo el esquema anterior, crea las carpetas necesarias.
  • Crea un nuevo sitio virtual a partir del ya existente y ajusta después la configuración.
  • Edita también el servidor DNS si es necesario.
  • Sube un documento HTML al sitio web.
  • Verifica el funcionamiento.
  • Usando la directiva ErrorDocument, muestra un error personalizado cuando el código de estado sea 404 Not Found.
  • Documenta los cambios realizados estructurando correctamente el contenido.

Actividad (parte 2): usuarios FTP

  • Crea un usuario específico para el sitio web con el nombre user_www.
  • Habilita acceso mediante FTP a la carpeta del sitio web para que puedan gestionar los archivos del sitio web mediante FTP.
  • Documenta los cambios realizados.

Recursos

En la Ubuntu Server Guide en el apartado Opciones básicas dispones de la información necesaria para realizar la instalación y la configuración.
