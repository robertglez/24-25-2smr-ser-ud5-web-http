Práctica 4: Control de acceso, autorización y autenticación

Control de acceso, autorización y autenticación

Objetivos

Actividades

Actividad 1: control de acceso

  1. Crea un directorio que responda a la siguiente URL http://www.xarxa00.lan/admin.
  2. Configura el nodo virtual para que solo se pueda acceder al directorio admin desde la red interna.
  3. Configura el nodo virtual para que cuando alguien intente acceder desde la red externa se muestre un error personalizado. Utiliza la directiva ErrorDocument.
  4. Realiza las capturas adecuadas para demostrar que funciona.

Actividad 2: autenticación básica

  1. Configura el nodo virtual para que el directorio admin requiera usuario para acceder.
  2. Crea el usuario mark con la contraseña 1234.
  3. Configura las directivas para que solo el usuario mark pueda acceder al directorio admin.
  4. Realiza las capturas adecuadas para demostrar que funciona.

Actividad

Actividad: Autenticación y autorización en Apache2
Los archivos .htaccess permiten a los usuarios que no tienen permisos modificar la configuración y así poder ejercer algún control sobre el comportamiento de su parte del servidor Apache2.

Las directivas que modifican el comportamiento se colocan en un archivo .htaccess situado en el directorio al que tiene que afectar, junto a todos sus subdirectorios. El archivo .htaccess se carga cada vez que se solicita un documento.

Fuente: https://httpd.apache.org/docs/trunk/es/howto/auth.html
